# MJRManager - Новые функции

## ✨ Добавленные возможности

### 1. 🔍 Глобальный поиск по содержимому
**Где:** Меню "Редактор" → "Глобальный поиск..." или кнопка "Поиск" на панели инструментов

**Горячая клавиша:** `Ctrl+Shift+F`

**Что делает:**
- Ищет текст во всех декомпилированных классах JAR файла
- Поддерживает регулярные выражения
- Опция учета регистра
- Показывает номер строки и контекст найденного совпадения
- Двойной клик по результату открывает файл

**Как использовать:**
1. Откройте JAR файл
2. Нажмите кнопку "Поиск" или `Ctrl+Shift+F`
3. Введите поисковый запрос
4. Выберите опции (регистр, regex)
5. Нажмите "Найти"
6. Дважды кликните на результат для открытия файла

---

### 2. 📄 Просмотр и редактирование манифеста
**Где:** Меню "JAR" → "Просмотр манифеста" / "Редактировать манифест" или кнопка "Манифест"

**Что делает:**
- Просмотр содержимого META-INF/MANIFEST.MF
- Редактирование манифеста с валидацией
- Изменение Main-Class и других атрибутов
- Автоматическое создание манифеста если его нет

**Как использовать:**
1. Откройте JAR файл
2. Нажмите кнопку "Манифест" или выберите в меню "JAR"
3. Для просмотра: выберите "Просмотр манифеста"
4. Для редактирования: выберите "Редактировать манифест"
5. Внесите изменения и нажмите "Сохранить"
6. Сохраните JAR файл (`Ctrl+S`)

---

### 3. ➕ Добавление и удаление файлов в JAR
**Где:** Меню "JAR" → "Добавить файлы..." / "Удалить выбранный файл"

**Что делает:**
- Добавление любых файлов в JAR (классы, ресурсы, конфиги)
- Удаление выбранных файлов из JAR
- Указание пути внутри JAR для новых файлов
- Автоматическое обновление дерева файлов

**Как использовать добавление:**
1. Откройте JAR файл
2. Выберите "JAR" → "Добавить файлы..."
3. Выберите один или несколько файлов
4. Укажите путь внутри JAR (например: `com/example/resources/`)
5. Файлы будут добавлены
6. Сохраните JAR (`Ctrl+S`)

**Как использовать удаление:**
1. Откройте JAR файл
2. Выберите файл в дереве (не пакет!)
3. Выберите "JAR" → "Удалить выбранный файл"
4. Подтвердите удаление
5. Сохраните JAR (`Ctrl+S`)

---

### 4. 📦 Экспорт в исходники
**Где:** Меню "Файл" → "Экспорт в исходники..."

**Горячая клавиша:** `Ctrl+E`

**Что делает:**
- Декомпилирует все классы в .java файлы
- Сохраняет структуру пакетов
- Экспортирует ресурсы и META-INF
- Создает README с инструкциями по компиляции
- Показывает прогресс экспорта

**Структура экспорта:**
```
output_folder/
├── src/              # Исходные коды (.java)
│   └── com/
│       └── example/
│           └── MyClass.java
├── resources/        # Ресурсы
├── META-INF/         # Манифест и метаданные
└── README.txt        # Инструкции
```

**Как использовать:**
1. Откройте JAR файл
2. Выберите "Файл" → "Экспорт в исходники..." или `Ctrl+E`
3. Выберите папку для экспорта
4. Дождитесь завершения (показывается прогресс)
5. Готово! Все исходники в выбранной папке

## 💡 Советы по использованию

1. **Глобальный поиск** работает параллельно - может занять время на больших JAR
2. **Манифест** валидируется перед сохранением - проверяйте формат
3. **Добавление файлов** - можно добавлять несколько файлов за раз
4. **Экспорт** - создает готовую структуру проекта с инструкциями
5. **Не забывайте сохранять JAR** после изменений (`Ctrl+S`)

---

## 🐛 Известные ограничения

1. Глобальный поиск может быть медленным на JAR с тысячами классов
2. Экспорт больших JAR может занять несколько минут
3. Некоторые обфусцированные классы могут не декомпилироваться корректно
